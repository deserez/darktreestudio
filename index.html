What this bot is for

Asheron is built around this server's staff workflow: setup, logs, tickets, suggestions, moderation history, welcome panels, temporary voice channels, and a few server games.

Members Use tickets, suggestions, reports, counting, lore commands, and temporary voice controls.
Staff Use moderation actions, history checks, notes, dashboard tools, announcements, and system panels.
Admins Use setup, health, config, welcome, counting setup, and dashboard commands to maintain the server.

Member channel

Commands members may see

These commands and panels are meant for normal server use. Some actions may be handled through buttons instead of typed commands.

/ticket

Private help

Creates ticket panels for support, reports, business, appeals, or other server needs.

Use when
You need staff help without posting in public channels.
What happens
Asheron creates a private thread, pings staff, and keeps controls for claiming, closing, reopening, and transcripts.

/suggest

Ideas

Posts a suggestion panel where members can submit ideas through a form.

Use when
You want to suggest a change, event, channel, rule, feature, or server improvement.
What happens
Asheron creates a tracked suggestion thread with voting and staff status controls.

/report

Safety

Sends a report to staff for review.

Use when
You need staff to look at a user, message, situation, or rule issue.
What happens
The report is tracked for staff, connected to history, and can be reviewed later.

/vc

Voice control

Manages temporary voice channels created for server members.

Use when
You own a temporary VC and need to lock, unlock, rename, limit, permit, remove, transfer, or delete it.
What happens
Asheron changes the channel permissions or settings for your current temp VC.

/counting

Game

Runs the server counting game.

Use when
You want to check the current count, leaderboard, or game status.
What happens
Asheron tracks numbers, mistakes, and leaderboard stats in the configured counting channel.

/asheron

Lore

Runs Asheron-themed interactions.

/asheron quote
Sends an Asheron directive.
/asheron signal
Starts the Signal Decode mini-game.

Staff channel

Moderation commands

These commands are for staff. They create records so future staff can understand what happened and why.

Command Meaning Use it when
/warn Records a warning for a member. A rule issue needs a formal note but not a timeout, kick, or ban.
/timeout Temporarily restricts a member from chatting. A situation needs immediate cooling down for a set duration.
/kick Removes a member from the server. The user can return later, but should be removed now.
/ban Bans a user and logs the reason. The user should not be able to rejoin.
/purge Deletes a batch of recent messages. A channel needs cleanup after spam, clutter, or rule-breaking messages.
/note Adds a private staff note to a member. Staff need context that should not be visible to the member.
/history Shows infractions, staff notes, tickets, reports, and suggestions. You need context before making a moderation decision.

Admin systems

Server maintenance commands

Use these when setting up or maintaining the server. They affect channels, panels, logs, and saved configuration.

/setup quick

Creates the core Asheron server structure: staff roles, logging channels, ticket areas, suggestion setup, dashboard pieces, and starter panels.

Run this first on a fresh server.

/health

Checks loaded commands, saved config, modules, and whether important Discord channels or assets can still be found.

Run this after setup or after moving channels.

/config view

Shows the current bot configuration so staff can see what Asheron believes is active.

Use this before troubleshooting.

/config backup

Exports the server configuration as JSON.

Use this before major server changes.

/dashboard post

Posts or refreshes the staff dashboard in the chosen channel.

Use this when staff tools need a clean control point.

/welcome rules

Posts a rules acknowledgement panel and can grant a role when members accept.

Use this in the welcome or rules channel.

/announce draft

Starts an editable announcement draft before posting it to an announcement channel.

Use this for clean staff-approved announcements.

/counting setup

Chooses the counting channel and starts the game state.

Use this before members begin counting.

Troubleshooting

FAQ

Why can I not see a command?

You may not have permission, the command may be staff-only, or slash commands may need to be deployed again.

What should admins run first?

Run /setup quick, then run /health. If anything is missing, fix that before posting member panels.

What should staff check before moderating?

Use /history when context matters. It helps connect reports, notes, infractions, tickets, and suggestions.

Is this bot meant for other servers?

No. This manual is written for our server's Asheron bot and its current workflow.
